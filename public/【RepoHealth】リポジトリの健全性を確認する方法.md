## はじめに

**[Repo Health](https://repo-health.up.railway.app/)** は、リポジトリの健全性を確認できるオープンソースツールです。

## できること

### 1. 総合スコア（0〜100点）

CHAOSS Metricsにもとづいた独自アルゴリズムでリポジトリを採点します。

```
スコア = (0.3 × Activity) + (0.25 × Maintenance) + (0.2 × Community) + (0.25 × Docs)
```

| 指標          | 重み | 内容                                             |
| ------------- | ---- | ------------------------------------------------ |
| Activity      | 30%  | コミット頻度・最新更新日・ユニーク作者数         |
| Maintenance   | 25%  | Issue対応速度・オープンIssue比率・リポジトリ年齢 |
| Community     | 20%  | スター数・フォーク数（対数スケール）             |
| Documentation | 25%  | README・LICENSE・CONTRIBUTINGの有無              |

さらに、GPT-4 Miniを使った**LLMによる±20点の補正**も行います。たとえば「6ヶ月コミットゼロ」のリポジトリでも、README解析で「安定した完成済みライブラリ」と判断した場合はスコアを加算します。

### 2. PRメトリクス分析

コントリビュート前に知りたい情報を可視化します。

- **マージ速度**: 平均マージ時間（数時間 vs 数ヶ月）
- **レビューの質**: ボットのみか、人間のレビュアーがいるか
- **コントリビューターの定着率**: サンキーダイアグラムで「初PR → 2回目 → 常連 → コアチーム」の流れを可視化

### 3. Issueの分析

- **平均クローズ時間**: 中央値も含めた実質的な対応速度
- **Hot Issues**: 直近48時間の活動・コメント数・セキュリティ関連キーワードで優先表示
- **Hidden Gems**: 長く放置されている高インパクトなIssue（初コントリビュートに最適）
- **Crackability Score**: ドキュメント品質・ファイル数・テスト要件から算出した難易度（0〜100）

### 4. プロジェクト構造の概観

リポジトリ全体のファイルツリーをLLMに渡し、**エントリーポイント・主要ファイル・各フォルダの役割**を自動で解説します。

加えて、**File-Issue Mapping**により、Issueの本文に記載されたファイルパスを正規表現で抽出し、該当ファイルと関連Issueをリンクします。「このファイルにどんな問題があるか」を一目で確認できます。

### 5. アクティビティパターン検出

スパムや異常なコミットを検出してリスクグレードを算出します。

| グレード | スコア  | 意味       |
| -------- | ------- | ---------- |
| A        | 0〜10   | 正常       |
| B        | 11〜30  | 軽微な異常 |
| C        | 31〜50  | 要確認     |
| D        | 51〜70  | 疑わしい   |
| F        | 71〜100 | 重大な問題 |

検出内容の例として、大量削除（コード変更の80%以上が削除）、短時間での連続コミット（10分以内に5件以上）などがあります。

### 6. コントリビューション Insights（メンテナー視点）

却下されたPRのパターンをLLMで分析し、「なぜ失敗したか」をカテゴリ別に表示します。

| カテゴリ | 意味                           |
| -------- | ------------------------------ |
| tests    | テスト不足・テスト失敗         |
| style    | コードフォーマット違反         |
| scope    | 変更範囲が広すぎる・スコープ外 |
| setup    | ビルド・環境の問題             |
| breaking | 破壊的変更の混入               |
| docs     | ドキュメント不足               |

READMEへの名前追加など明らかなスパムPRは自動フィルタリングされます。

## 使い方

1. [デモサイト](https://repo-health.up.railway.app/)にアクセス
2. 分析したいGitHubリポジトリのURLまたは `owner/repo` 形式を入力
3. 各タブで「スコア」「PRメトリクス」「Issue分析」「プロジェクト構造」などを確認

プライベートリポジトリも分析可能（GitHubでサインイン後）。キャッシュはトークンハッシュで分離されており、他ユーザーのプライベートデータは参照できません。
