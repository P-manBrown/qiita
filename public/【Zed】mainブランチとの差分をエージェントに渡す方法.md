## はじめに

Zedエディタのバージョンアップにより、AIエージェントとのやり取りがさらに便利になりました。
2026年3月10日にマージされたPR #51235 によって、**現在のブランチとmainブランチとの差分を、エージェントへ `@` メンションで直接渡せる機能**が追加されました。

https://github.com/zed-industries/zed/pull/51235

この記事では、その使い方と活用シーンをわかりやすく解説します。

## この機能で何ができるのか

以前から、ZedにはAIエージェント（Agent Panel）にファイルやコードをコンテキストとして渡す機能がありました。
今回の追加機能では、**`@branch diff` とタイピングするだけで、現在のブランチとmainブランチとの差分全体**をエージェントのコンテキストに含められるようになりました。

これにより、たとえば次のような場面で役立ちます。

- 「このPRで何を変えたか確認して」とエージェントにレビューを依頼する
- 変更内容をもとにコミットメッセージやPR説明文を自動生成してもらう
- 差分をもとにテストコードを書いてもらう

## 使い方

### 1. Zedでプロジェクトを開く

まずZedでGitリポジトリが設定されたプロジェクトを開きます。
作業ブランチに切り替えておいてください（例: `feature/add-login`）。

### 2. Agent Panelを開く

`Cmd + ?`（macOS）または `Ctrl + ?`（Linux / Windows）でAgent Panelを開きます。

あるいは、画面右下のエージェントアイコンをクリックしてもOKです。

### 3. `@branch diff` をメンションする

Agent Panelのテキスト入力欄で `@` と入力し、続けて `branch diff` とタイピングします。

```
@ branch diff
```

入力すると候補として `Branch Diff with Main` が表示されます。
これを選択すると、現在のブランチとmainとの差分がコンテキストとして追加されます。

> **ヒント**: `branch diff` というキーワードで**ファジーマッチ（あいまい検索）**が働くため、多少タイポしても候補が表示されます。

### 4. エージェントへ指示を送る

差分がコンテキストに追加された状態で、エージェントへ自由に指示を送れます。

```
@BranchDiff このPRで変更した内容をもとに、Pull Requestの説明文を日本語で書いてください。
```

```
@BranchDiff この差分にバグや潜在的な問題がないかレビューしてください。
```

## 主なユースケース

### PRの説明文を自動生成する

差分をエージェントに渡して、Pull Requestの本文を自動生成させることができます。

```
@BranchDiff 以下の観点でPull Requestの説明文を書いてください。
- 変更の概要
- 変更した理由
- テスト方法
```

### コードレビューを依頼する

```
@BranchDiff このコード変更をレビューして、改善点や問題点を指摘してください。
```

### コミットメッセージを提案してもらう

```
@BranchDiff この変更内容に合ったコミットメッセージをConventional Commits形式で提案してください。
```

## 注意点

### mainブランチが無効な場合は使えない

PRの実装では、**mainブランチが存在しない・無効な状態の場合、`@branch diff` はメニューに表示されません**。
リポジトリにmainブランチが設定されていることを確認してください。

`master` ブランチをデフォルトブランチとして使用しているプロジェクトでは、現時点では差分の参照先が `main` に固定されているため注意が必要です。

### 差分が大きいとコンテキストを圧迫する可能性がある

変更量が非常に多いブランチの場合、差分全体をコンテキストに渡すとトークン数が増え、レスポンスの精度が落ちたり応答が遅くなることがあります。
その場合は、特定のファイルだけを `@file` でメンションして絞り込むことを検討してください。
