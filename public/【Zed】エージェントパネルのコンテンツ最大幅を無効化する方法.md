## はじめに

Zed 0.233.5 のリリースで、エージェントパネルに `limit_content_width` という新しい設定が追加されました。

この設定を使うと、エージェントパネルのコンテンツに適用されていた最大幅の制限を解除できます。ワイドモニターを使っている方や、エージェントパネルを広い幅で使いたい方に役立つ機能です。

## 背景：なぜ最大幅の制限があるのか

デフォルトでは、Zed のエージェントパネルのコンテンツは一定の最大幅に制限されています。これは、テキストの1行が長くなりすぎると読みにくくなるという考え方（いわゆる「行長の最適化」）に基づいています。

しかし、広いモニターでエージェントパネルを大きく表示している場合、コンテンツが中央に固まって両端に余白が生じるため、画面領域を無駄に感じることがあります。

今回追加された `limit_content_width` 設定により、この制限を自分の好みに合わせて変更できるようになりました。

## 設定方法

`settings.json` に以下のように記述します。

```json
{
  "agent": {
    "limit_content_width": false
  }
}
```

`false` に設定すると、コンテンツの最大幅制限が無効になり、パネルの幅いっぱいにコンテンツが表示されます。

### settings.json の開き場所

`Cmd + ,`（macOS）または `Ctrl + ,`（Linux / Windows）でユーザー設定ファイルを開けます。

または、コマンドパレット（`Cmd + Shift + P` / `Ctrl + Shift + P`）から `zed: open settings` を実行しても開けます。

## デフォルト値について

`limit_content_width` のデフォルト値は `true`（最大幅制限あり）です。

明示的に設定しない場合は、従来どおりコンテンツに最大幅の制限がかかった状態になります。

## 設定値まとめ

| 設定値 | 動作 |
|--------|------|
| `true`（デフォルト） | コンテンツの最大幅を制限する |
| `false` | コンテンツの最大幅制限を無効にする |

## 活用シーン

この設定が特に役立つのは以下のようなケースです。

エージェントパネルをワイドな状態で使いたい場合、画面の横幅を最大限活用してコンテンツを表示できます。また、コードブロックが長い場合に折り返しを減らして見やすくしたい場合にも有効です。逆に、読みやすさを重視する場合はデフォルトの `true` のままにしておくことをおすすめします。
