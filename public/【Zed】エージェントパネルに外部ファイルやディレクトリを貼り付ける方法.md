## はじめに

Zed のエージェントパネルでは、プロジェクト内のファイルを `@` メンションでコンテキストとして追加できます。しかし、プロジェクト外にあるファイルやディレクトリをコンテキストとして渡したい場面もあります。

PR #52300 でエージェントパネルへのファイル・フォルダの貼り付けがサポートされ、外部のファイルやディレクトリもエージェントのコンテキストとして手軽に渡せるようになりました。

本記事では、その具体的な方法をまとめます。

## 対象読者

- Zed エディタを使っている方
- エージェントパネルでファイル操作を多用する方
- プロジェクト外のファイルを AI に参照させたい方

## 方法1: ファイル・フォルダをドラッグ＆ドロップする

OS のファイルマネージャー（Finder や Explorer など）からファイルやフォルダをドラッグして、エージェントパネルのメッセージ入力欄にドロップします。

これだけで、そのファイルやディレクトリがコンテキストとして `@` メンション形式で追加されます。

```
# 操作手順
1. Finder / Explorer などでファイルまたはフォルダを選択
2. エージェントパネルのメッセージ入力欄にドラッグ
3. ドロップするとコンテキストとして追加される
```

## 方法2: ファイルパスをコピーして貼り付ける

ファイルやディレクトリのパスをクリップボードにコピーしてから、エージェントパネルのメッセージ入力欄に貼り付けます。

Zed はパスを検出してファイルやフォルダとして認識し、`@` メンション形式に変換します。

```
# macOS でのパスコピー例
# Finder でファイルを右クリック →「パス名をコピー」
# または以下のコマンドでパスを取得してコピー
$ pwd | pbcopy
$ echo /path/to/your/file.txt | pbcopy
```

```
# Windows でのパスコピー例
# Explorer でファイルを Shift + 右クリック →「パスのコピー」
```

コピーしたパスをエージェントパネルに `Cmd+V`（Windows は `Ctrl+V`）で貼り付けると、Zed が自動的にファイルまたはフォルダとして処理します。

## 方法3: エディタバッファから複数行をコピーして貼り付ける

Zed のエディタ内でコードを選択してコピーし、エージェントパネルに貼り付けた場合、Zed は自動的にそのコードをファイルコンテキスト付きの `@` メンション形式にフォーマットします。

```
# 操作手順
1. エディタでコードブロックを選択（複数行）
2. Cmd+C / Ctrl+C でコピー
3. エージェントパネルのメッセージ入力欄に Cmd+V / Ctrl+V で貼り付け
4. ファイル名付きの @メンション形式で追加される
```

フォーマットなしで生テキストとして貼り付けたい場合は `Cmd+Shift+V`（Windows は `Ctrl+Shift+V`）を使います。

## 方法4: セレクションをコンテキストとして追加する

エディタやターミナルでテキストを選択した状態で、以下のいずれかの操作を行うと、選択内容をスレッドのコンテキストとして追加できます。

- キーバインド `Cmd+>` / `Ctrl+>` を押す
- コマンドパレットで `agent: add selection to thread` を実行する
- メッセージ入力欄の `+` メニューから「Selection」を選択する

```
# キーバインドの例
# macOS
Cmd + >

# Windows / Linux
Ctrl + >
```

## コンテキスト追加方法のまとめ

| 方法 | 操作 | 用途 |
|---|---|---|
| ドラッグ＆ドロップ | ファイル・フォルダをパネルへドロップ | プロジェクト外のファイル |
| パスを貼り付け | パスをコピー → Cmd+V | ファイル・ディレクトリのパス指定 |
| バッファからコピー | コード選択 → コピー → 貼り付け | エディタ内のコード断片 |
| セレクション追加 | テキスト選択 → Cmd+> | エディタ・ターミナルの選択範囲 |
| @メンション | 入力欄で @ を入力 | プロジェクト内のファイル・ディレクトリ |

## @メンションによる従来の方法との使い分け

メッセージ入力欄で `@` を入力するとファイル・ディレクトリ・シンボル・過去スレッドなどをコンテキストとして検索・追加できます。ただし、この方法はプロジェクト内のファイルが対象です。

プロジェクト外のファイルやディレクトリを参照させたい場合は、上述のドラッグ＆ドロップやパス貼り付けの方法を使うのが適しています。

## 画像のコンテキスト追加（補足）

ビジョンモデルを使用している場合は、画像をエージェントパネルに貼り付けることもできます。対応しているプロバイダは以下の通りです。

- OpenAI GPT-4o 以降
- Anthropic Claude 3 以降
- Google Gemini 1.5 / 2.0
- Bedrock のビジョンモデル（Claude 3+、Amazon Nova Pro/Lite、Meta Llama 3.2 Vision、Mistral Pixtral）

画像の追加はドラッグ＆ドロップまたはクリップボードからの貼り付けで行えます。

## まとめ

- エージェントパネルにはファイル・ディレクトリをドラッグ＆ドロップまたはパス貼り付けで追加できる（PR #52300 で追加）
- バッファからのコピー貼り付けは自動的に `@` メンション形式に変換される
- `Cmd+>` でエディタやターミナルの選択範囲をコンテキストとして追加できる
- プロジェクト内のファイルは `@` メンション、プロジェクト外はドラッグ＆ドロップやパス貼り付けで使い分けるとよい
